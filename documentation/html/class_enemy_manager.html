Detailed Description

Enemy manager class. Keeps all enemies inside a vector of unique pointers Is responsible for calling Enemy::process(Player&), Enemy::render(sf::RenderWindow&), receiving new enemies and cleaning up the disabled enemies. Has exclusive ownership over the stored objects.

◆ process()

void EnemyManager::process ( Player & player,
ProjectileManager & projectiles,
PickupManager & pickups )

Main process function. Is to be called each frame by the game controller. It is resposible for calling cleanupEnemies(),checkWave() and calling Enemy::process(Player&) for each non-disabled enemy and checking collisions with player projectiles.

Parameters
player- a reference to the Player object, necessary to pass as a parammeter to Enemy::process()
projectiles- a reference to the ProjectileManager, necessary to check for collisions.
pickups- a reference to the PickupManager, necessary for enemies to create pickups on destruction.
See also
EnemyManager::cleanupEnemies, EnemyManager::checkWave, Enemy::process, CollisionCircle::checkCollision
