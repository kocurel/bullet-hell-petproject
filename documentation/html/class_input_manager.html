Detailed Description

The InputManager class handles keyboard input states.

This class implements the Singleton design pattern to ensure that there is only one instance of the input manager throughout the application. It tracks which keys were just pressed in the current frame

void InputManager::clearInputs ( )

Clears all 'just pressed' states for all keys.

This method should be called once at the end of each game loop iteration (after all events have been processed and input queries have been made) to prepare for the next frame.

◆ isKeyJustPressed()

bool InputManager::isKeyJustPressed ( sf::Keyboard::Key key) const

Checks if a key was just pressed in the current frame.

A key is considered 'just pressed' if it transitioned from not pressed to pressed within the current frame. This state is reset by clearInputs() at the end of each frame.

Parameters
keyThe SFML keyboard key code to check.
Returns
True if the key was just pressed, false otherwise.

◆ setKeyJustPressed()

void InputManager::setKeyJustPressed ( sf::Keyboard::Key key)

Sets the state of a key to 'just pressed'.

This method should be called when an sf::Event::KeyPressed event is detected. The key's 'just pressed' state will be true until clearInputs() is called.

Parameters
keyThe SFML keyboard key code (e.g., sf::Keyboard::Z, sf::Keyboard::Space).
